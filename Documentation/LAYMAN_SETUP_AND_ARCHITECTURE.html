Driver License Platform - Setup & Architecture Guide

For Non-Technical Users

üìñ What is This Application?

This is a Driver License Platform - a website that helps people prepare for their driver's license test and book appointments at the DMV (Department of Motor Vehicles).

Think of it like this:

  • Practice Tests: Like a study guide app on your phone, but for driver's license tests
  • Appointment Booking: Like booking a restaurant reservation, but for DMV visits
  • Multi-State Support: Works for California, Texas, Florida, and New York

üèóÔ∏è How Does It Work? (Simple Explanation)

The Three Main Parts:

1. The Frontend (What You See)

This is like the storefront of a shop. It's what you see and interact with on your computer or phone. Built with React - a popular tool for building websites.

  • Login and registration pages
  • Practice test interface
  • Appointment booking calendar
  • Your personal dashboard

2. The Backend (The Brain)

This is like the kitchen of a restaurant - you don't see it, but it does all the work. Built with Node.js and Express.

  • Stores your account information
  • Saves your test scores
  • Manages your appointments
  • Handles security (passwords, login)

3. The Database (The Filing Cabinet)

This is like a filing cabinet that stores all information. Uses MongoDB - a modern database system.

  • User accounts and passwords
  • Practice test questions and answers
  • Appointment records
  • Test results and history

üîÑ How They Work Together

Imagine ordering food at a restaurant:

  1. You (Frontend): You look at the menu and place an order
  2. Waiter (Backend): Takes your order to the kitchen
  3. Kitchen (Database): Prepares and stores the information
  4. Waiter (Backend): Brings the food (data) back to you
  5. You (Frontend): See and enjoy your meal (the results)

üíª Setting Up the Application (Step-by-Step)

Step 1: Install Required Software

Think of this like installing apps on your phone. You need:

  • Node.js: The engine that runs the application
    Download from: https://nodejs.org/
    Choose the LTS (Long Term Support) version - it's the most stable
  • Git: A tool to manage code and upload to GitHub
    Download from: https://git-scm.com/download/win

Step 2: Install Application Dependencies

Dependencies are like ingredients needed to cook a meal. The application needs many small pieces of code (libraries) to work properly.

This happens automatically when you run:

Backend: npm install
Frontend: cd client then npm install

This may take 2-5 minutes - be patient!

Step 3: Configure Environment

Create a .env file - this is like a settings file that tells the application:

  • Where to find the database (MongoDB)
  • Security keys (like a password for the application)
  • Which port to run on (like a door number)

Step 4: Start the Servers

You need to start TWO servers (like opening two apps):

Server Command What It Does
Backend npm run dev Starts the brain of the application
Frontend cd client
npm run dev		 Starts the user interface

Step 5: Access the Application

Open your web browser and go to:

http://localhost:3000

localhost means "this computer" and 3000 is the port number (like a door number).

üèõÔ∏è Architecture Overview (Simple Terms)

The Flow of Information

  1. User opens browser ‚Üí Goes to http://localhost:3000
  2. Frontend loads ‚Üí Shows login page, dashboard, etc.
  3. User clicks "Take Test" ‚Üí Frontend asks Backend for test questions
  4. Backend checks Database ‚Üí Gets test questions
  5. Backend sends to Frontend ‚Üí Questions appear on screen
  6. User answers questions ‚Üí Frontend sends answers to Backend
  7. Backend calculates score ‚Üí Saves to Database
  8. Frontend shows results ‚Üí User sees their score

Security Features

  • Password Hashing: Passwords are scrambled before storing (like a secret code)
  • JWT Tokens: Like a temporary ID card that proves you're logged in
  • CORS: Security rules about who can talk to the server
  • Rate Limiting: Prevents too many requests (like limiting how many times you can knock on a door)

üì¶ Deployment Options

1. Local Development (On Your Computer)

Running on your own computer for testing. Like having a restaurant in your own kitchen.

  • Fast to set up
  • Only you can access it
  • Good for development and testing

2. Docker (Containerized)

Packages everything into a "container" - like a shipping box with everything inside.

  • Easy to move between computers
  • Consistent environment
  • Includes database in the same container

3. Kubernetes (Cloud Production)

Deploys to the cloud with automatic scaling. Like having multiple restaurants that open more locations when busy.

  • Handles high traffic automatically
  • Multiple servers for reliability
  • Professional production setup

üîß Troubleshooting Common Issues

"Site can't be reached" Error

  • Check if both servers are running (look for PowerShell windows)
  • Wait 10-15 seconds after starting servers
  • Try: http://localhost:3000 (not https://)
  • Check firewall settings

Port Already in Use

  • Another application is using port 3000 or 5000
  • Close other applications or change ports in configuration

MongoDB Connection Error

  • MongoDB might not be running
  • Check connection string in .env file
  • For testing, some features work without MongoDB
