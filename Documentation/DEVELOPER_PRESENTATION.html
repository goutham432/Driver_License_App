Authentication Flow

Registration

  1. User submits form → POST /api/auth/register
  2. Backend validates & hashes password (bcrypt, 10 rounds)
  3. Save user to MongoDB
  4. Generate JWT token (expires 7 days)
  5. Return token + user data

Login

  1. POST /api/auth/login with email/password
  2. Verify password with bcrypt.compare()
  3. Generate & return JWT token

Protected Routes

  1. Frontend adds Authorization: Bearer <token>
  2. Middleware verifies JWT with secret
  3. Attach user to req.user

Database Schema

Collection Key Fields Indexes
users email (unique), password (hashed), state, role email, state
tests state, category, questions[], passingScore state, category
appointments user (ref), state, date, timeSlot, status user, date, state

Relationships: Appointments reference Users (ObjectId), Users store test scores array with testId references

Docker Deployment

Multi-stage Build

# Stage 1: Build React frontend FROM node:20-alpine AS frontend-builder WORKDIR /app/client COPY client/package*.json ./ RUN npm install COPY client/ . RUN npm run build # Stage 2: Final image FROM node:20-alpine WORKDIR /app COPY --from=frontend-builder /app/client/dist ./client/dist COPY package*.json ./ RUN npm install --production COPY . . EXPOSE 5000 CMD ["node", "server.js"]

Benefits: Smaller image size, optimized production build, single container deployment

Kubernetes Deployment

Components
